## Design Guidelines

### Mobile-First Approach
- Default styles for mobile, use `md:` and `lg:` for larger screens
- Touch targets minimum 44x44px
- Readable font sizes (min 14px/0.875rem for body)
- Adequate spacing (min 16px/1rem between interactive elements)

### Breakpoints to Use
```
sm: 640px   - Small tablets, large phones
md: 768px   - Tablets
lg: 1024px  - Small laptops
xl: 1280px  - Desktops
```

### Tailwind Classes Reference
- Responsive: `md:hidden`, `hidden md:block`, `sm:grid-cols-2`
- Touch targets: `min-h-11 min-w-11` (44px)
- Spacing: `gap-4`, `p-4`, `px-4 py-3`
- Text: `text-sm md:text-base`, `text-2xl md:text-3xl`

## Implementation Notes

1. **Mobile Menu Animation:**
   - Use Framer Motion (already in project) for smooth slide-in
   - Or simple CSS transition with transform
   - Backdrop blur for overlay

2. **Icon Button Accessibility:**
   - All icon buttons must have aria-label
   - Consider adding text labels on mobile if space permits

3. **Performance:**
   - Use `will-change-transform` sparingly
   - Prefer CSS transitions over JS animations where possible

4. **Existing Patterns to Follow:**
   - Use existing color scheme (slate-950 bg, cyan-400 accents)
   - Follow existing border styles (border-white/10, rounded-2xl)
   - Use backdrop-blur for glass effects
   - Keep hover effects for desktop, remove/adapt for touch
